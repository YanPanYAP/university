**12. Понятие *in-memory Database***
*In-memory database* — это СУБД, в которой основные данные хранятся в оперативной памяти (RAM), а не на диске. Это обеспечивает минимальную задержку доступа и высокую пропускную способность операций чтения/записи. Дисковое хранилище может использоваться для персистентности, но рабочий набор данных обрабатывается в памяти.

Типичный пример — Redis. Такие системы применяются для кэширования, сессий, счетчиков, real-time аналитики и очередей сообщений.

---

**13. Понятие хэш-таблица**
Хэш-таблица — это структура данных, обеспечивающая доступ к значениям по ключу за амортизированное время O(1). Она использует хэш-функцию, которая преобразует ключ в индекс массива (бакета).

При коллизиях (совпадении индексов) применяются методы разрешения: цепочки (separate chaining) или открытая адресация (linear probing и др.). В Redis хэш-таблицы используются для реализации словарей (key–value) и структуры данных типа hash.

---

**14. Персистентность данных в Redis**
В Redis персистентность обеспечивается двумя механизмами:

1. **RDB (Redis Database Snapshotting)** — периодическое создание снимков (snapshot) базы данных и сохранение их в файл dump.rdb.
2. **AOF (Append Only File)** — журналирование всех операций записи (write-ahead log), позволяющее восстановить состояние БД путём повторного воспроизведения команд.

Возможна совместная настройка RDB + AOF для повышения надежности.

---

**15. Назначение команд: `set`, `get`, `del`, `getset`**

* `SET key value` — установить значение строкового ключа.
* `GET key` — получить значение по ключу.
* `DEL key` — удалить ключ и его значение.
* `GETSET key value` — установить новое значение и вернуть старое (атомарная операция).

Применяются для базовой работы со строковыми типами данных.

---

**16. Назначение команд: `incr`, `decr`**

* `INCR key` — увеличить числовое значение ключа на 1.
* `DECR key` — уменьшить числовое значение ключа на 1.

Если ключ отсутствует, он создаётся со значением 0 перед операцией. Команды атомарны и используются для счетчиков (например, количество просмотров).

---

**17. Назначение команд: `mset`, `mget`**

* `MSET key1 value1 key2 value2 ...` — установить несколько значений за одну операцию.
* `MGET key1 key2 ...` — получить значения нескольких ключей одновременно.

Позволяют уменьшить сетевые накладные расходы за счёт пакетной обработки.

---

**18. Назначение команд: `hset`, `hget`**

* `HSET key field value` — установить значение поля `field` внутри хэша `key`.
* `HGET key field` — получить значение конкретного поля хэша.

Используются для хранения структурированных данных (например, объекта пользователя).

---

**19. Назначение команд: `hmset`, `hmget`**

* `HMSET key field1 value1 field2 value2 ...` — установить несколько полей хэша.
* `HMGET key field1 field2 ...` — получить значения нескольких полей хэша.

Позволяют эффективно работать с множеством атрибутов объекта.

---

**Команда `EXISTS`**

* `EXISTS key` — проверяет существование ключа.
  Возвращает `1`, если ключ существует, и `0`, если нет.

Используется для проверки наличия данных перед выполнением операций.
